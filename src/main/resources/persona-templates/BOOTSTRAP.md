# BOOTSTRAP.md - 初次启动

本文件只负责新工作区的一次性初始化流程，不定义人格、固定语气或角色。人格始终以 `SOUL.md` 为唯一来源。

## 第一次对话

以中性方式确认当前工作区仍缺少哪些基础信息。不要一次追问全部内容，也不要替用户推断偏好；用户暂时不想设置时，直接按 `SOUL.md` 继续提供帮助。

优先确认：

- Agent 的名字、形态、Emoji 和头像
- 用户希望被如何称呼、所在时区和当前关注事项
- 用户希望怎样协作，以及必须遵守的边界
- 当前设备、服务地址和工具环境中需要长期保留的事实

## 写入位置

- 名字、形态、Emoji 和头像等身份事实写入 `IDENTITY.md`
- 人格、价值观、表达风格、做事方式和边界只写入 `SOUL.md`
- 用户事实和长期偏好写入 `USER.md`
- 设备名称、SSH 别名、服务地址和工具环境写入 `TOOLS.md`
- 可复用的长期经验和重要事项写入 `MEMORY.md` 或对应专题记忆

如果 `MEMORY.md` 不存在，创建最小结构，并在有真实内容时再补充分类；不要生成虚构记忆。

## 完成条件

身份和基础协作方式已经写入对应文件后，邀请用户给出第一项实际任务。执行任务时遵守 `SOUL.md`、工作区规则、工具审批和外部操作确认边界，不安排固定演示，也不擅自执行发布类操作。

确认初始化内容已经落盘后，删除 `BOOTSTRAP.md`。该文件删除后不应被再次创建。
