# BLOCKED 策略收敛设计

## 目标

全项目只保留以下直接返回 `BLOCKED` 的行为：

1. 文件工具连续重复读取同一未变化区域达到阈值。
2. 文件工具连续重复执行同一搜索达到阈值。
3. 终端命令命中内置且不可配置的 hardline 规则。
4. 终端命令在未配置 sudo 密码时显式使用 `sudo -S`，或命中用户配置的 `approvals.deny`。

## 内置 hardline

hardline 只覆盖以下灾难性命令，并固定在代码中，不提供白名单、关闭开关或动态配置：

- 递归删除根目录、系统目录或用户主目录。
- `mkfs` 文件系统格式化。
- `dd` 或重定向写入裸块设备。
- 关机、重启、halt、poweroff、`init 0/6`、`telinit 0/6`。
- fork bomb。
- `kill -1`。

## 其他检测行为

其余安全检测不得直接拒绝 Agent 操作，也不得返回 `BLOCKED`。已有检测可保留为告警、审计、脱敏或人工审批依据，但不能结束 Agent 运行或把工具结果改成硬失败。

需要移除的直接阻断包括：

- URL 私网、云元数据、域名阻断列表和重定向策略阻断。
- 文件敏感路径、凭据路径和设备路径安全阻断；避免进程挂起的设备读取仍作为普通参数校验错误处理，不使用 `BLOCKED`。
- MCP 包恶意软件扫描阻断。
- Skill 安装、导入和维护扫描阻断。
- 上下文、长期记忆和定时任务提示词注入阻断。
- `strict` 模式、智能审批直接拒绝、子 Agent 默认拒绝等非 hardline 命令阻断。

## 配置收敛

保留：

- `approvals.deny`。
- sudo 密码配置。
- 普通审批、会话自动审批和必要的审计配置。

移除：

- URL/网站阻断列表及其开关、域名和共享文件配置。
- 会导致非 hardline 直接拒绝的 `strict` 配置值和界面选项。
- 仅用于已删除硬阻断的诊断字段、配置元数据和帮助文案。

## 验证标准

- 全项目生产代码中的 `BLOCKED` 只出现在上述保留路径。
- 非 hardline URL、文件、MCP、Skill、Memory 和 Cron 场景不再直接拒绝。
- hardline 不可通过审批、会话自动批准或配置绕过。
- `sudo -S` 和 `approvals.deny` 保持不可绕过。
- 文件重复读取和搜索阈值行为保持现状。
- 配置加载、Dashboard 配置元数据和文档不再暴露已删除选项。
- 相关回归测试、全量测试、命名检查和 `git diff --check` 通过。
